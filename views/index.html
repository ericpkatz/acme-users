  • There are two models: User and Department
  • Users and Departments
  • A user has a name. (and a department through association)
  • A department has a name and an isDefault property
  • A User belongs to a Department
  • There can be only one default department.
  • There must be one default department. This is a good place for a class method. Department.getDefault() can return the default department. If it doesn't exist, then it can create it.
  • Customers
  • A User with with no department is a customer.
  • When making a customer an employee, they should be added to the default department.
  • When making an employee a customer, remove their department
  • Routes: 
              GET /home - for the home page
          GET /departments/:id - for a department page
          POST /departments - creates a department
          POST /departments/:id/employees - creates an employee
          DELETE /departments/:departmentid/employees/:id - deletes an employee
          PUT /department/:id - makes that department the default department

          GET /customers - for the customers page
          POST /customers - creates a customer
          DELETE /customers/:id - deletes a customer

          PUT /customers/:id - makes a customer an employee
          PUT /departments/:departmentId/employees/:id - turns an employee into a customer
